# 2026-05-05 — HNH pages.js IIFE Fix + Server-Side Grid Pre-Render

## Root Cause (resolved)
The `scr()` function in `hnh-portal/src/pages.js` was injecting `var DE/BR/DO` JSON arrays
into the browser and building DOM grids client-side via `rd()`, `rb()`, `rdo()`.
The `.map()` callbacks used `\"` inside single-quoted source strings — which produces literal `"`
in the output script — breaking string delimiters mid-expression → `SyntaxError`, IIFE crash.
Same bug in `loadIntegrations()` html+= line and nphies_mirror stats inject.

## What Was Fixed

### hnh-portal/src/pages.js
1. **`hdr()` book button**: `href="tel:+p"` → `href="https://bsma.elfadil.com" target="_blank"`
   (Header "Book Appointment" now opens BSMA portal directly instead of dialing)

2. **`scr()` full rewrite** (server-side pre-render strategy):
   - Removed `var DE=...`, `var BR=...`, `var DO=...` JSON injections
   - Removed `rd()`, `rb()`, `rdo()` functions and their `rd();rb();rdo();` init calls
   - Fixed `loadIntegrations()`:
     - Error/offline innerHTML: `\"` → `\'` pattern (HTML attrs in single-quoted JS strings)
     - `html+=` forEach line: rebuilt as intermediate vars (`cls`, `scls`, `stxt`, `tnl`)
       then `html+=\'<div class="\'+cls+\'">...\'` — valid, no ambiguous quote nesting
     - `nphies_mirror` stats inject: same `\'...\'+expr+\'...\'` fix
   - `doSearch()` was already correct (used `\'` for HTML attrs) — left unchanged
   - All other functions (scroll, lang toggle, IntersectionObserver, stats) unchanged

3. **`servePage()` — server-side grid rendering**:
   - Added `dgHtml`, `bgHtml`, `dogHtml` pre-render vars (pure server-side JS, no quoting issues)
   - Sections now inject HTML directly: `<div class="g4" id="dg">'+dgHtml+'</div>`
   - Instant first-paint for departments, branches, doctors — no JS required
